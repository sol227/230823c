1. clone : 로컬에 저장소가 없는 경우. 저장소 자체 복사
$ git clone URL주소 - 내용 수정 후 commit - push

2. pull : 로컬에 저장소가 있는 경우. 저장소 안에 파일 내려받기
$ git pull origin main -
- 온라인에서 수정하고 바로 로컬에서 수정하면 충돌이 일어남
- 충돌시 해결 방법 : 하나의 문서가 온라인과 로컬 각각 수정이 일어났기 때문에
  -> pull로 온라인에서 로컬로 내려받음 - 받은 로컬의 문서 수정 후 다시 commit - push로 온라인 업로드

3. 풀 리케스트 : 포크로 가져온 레파지토리를 수정해 원작자에게 수정 반영을 요청
- fork - clone - 내용 수정 후 commit - push
